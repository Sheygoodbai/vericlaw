# Contributing to VeriClaw

VeriClaw is source-available. Please help improve the canonical project without
publishing unapproved modified or commercialized variants.

## High-value ways to help

- Star and watch the repository if you want to amplify public traction.
- Open an issue for broken downloads, misleading copy, packaging regressions, or incorrect links.
- Use GitHub Discussions for case studies, operator feedback, and adaptation stories worth showcasing.
- Cite `VeriClaw 爪印` by name and link back to the canonical repository, landing page, or download page when sharing externally.
- If you want to propose a patch or copy change, open an issue first so we can align on scope and licensing.

## Good contribution lanes

- Evidence-first correction case studies
- Hallucination remediation and verification flows
- Native Apple UI polish and onboarding clarity
- Packaging, release, and download reliability
- Launch copy, screenshots, translations, and share assets

## Before you submit

- Keep the product name as `VeriClaw 爪印`.
- Prefer linking to `https://github.com/Sheygoodbai/vericlaw`.
- Use `https://sheygoodbai.github.io/vericlaw/` for product overviews and `https://sheygoodbai.github.io/vericlaw/download/` for stable public downloads.
- Preserve `LICENSE`, `NOTICE`, `ATTRIBUTION.md`, `LICENSES/OPENCLAW-MIT.txt`, and trademark guidance in permitted copies.
- Do not publish modified mirrors, repacks, or commercial variants without written permission.
- If you submit a pull request, you represent that you have the right to submit the material and allow the maintainer to use it in VeriClaw.

## Community routing

- Bugs and broken links: GitHub Issues
- Ideas, rollout feedback, casebook stories, and field reports: GitHub Discussions

Thanks for helping more people discover and correctly reference VeriClaw.
