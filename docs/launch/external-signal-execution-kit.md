# External signal execution kit

Execution date:
`2026-04-06`

This file turns the existing launch assets into a practical external-signal
plan for `VeriClaw 爪印`.

The goal is not fake virality.
The goal is to create real signals that search engines, directories, review
sites, and recommendation systems can actually see:

- indexed URLs
- external citations
- branded searches
- referral clicks
- durable backlinks
- repeated mention of the same canonical wording

## What counts as an external signal

Strong signals:

- a third-party page links to a canonical VeriClaw route
- a search engine receives the sitemap or page URL directly
- someone searches `VeriClaw` or `clawhub install vericlaw`
- a review site uses the official wording instead of inventing a bad summary
- a directory classifies VeriClaw under correction or supervision rather than a generic chatbot bucket

Weak signals:

- more on-site pages with no outside references
- vague social impressions with no clicks
- mentions that do not include the `VeriClaw` name
- links to mirrors or reposts instead of canonical routes

## Priority order

### Tier 1: search engine notification

Do these first because they are the cleanest, lowest-friction visibility
signals.

1. `Google Search Console`
   - property: `https://sheygoodbai.github.io/vericlaw/`
   - submit sitemap:
     `https://sheygoodbai.github.io/vericlaw/sitemap.xml`
   - request indexing for:
     - landing page
     - review kit
     - directory submission kit
     - recommendation snippets
     - new long-tail pages

2. `Bing Webmaster Tools`
   - property: `https://sheygoodbai.github.io/vericlaw/`
   - submit sitemap:
     `https://sheygoodbai.github.io/vericlaw/sitemap.xml`
   - request indexing for the same priority pages

3. `IndexNow` if you want faster Bing-adjacent pickup
   - use only canonical public URLs
   - submit the new long-tail pages and directory kit pages first

Official entry points:

- Google Search Console:
  `https://search.google.com/search-console`
- Bing Webmaster Tools:
  `https://www.bing.com/webmasters/about`
- IndexNow:
  `https://www.indexnow.org/`

Success signal:

- the URLs appear as indexed or discovered in the relevant webmaster tools

## Tier 2: citation surfaces

These are the highest-value external surfaces after search engines because they
create reusable backlinks and recommendation context.

Target surfaces:

- AI tool directories
- agent tool directories
- MCP-adjacent directories that allow non-server companion tools
- OpenClaw ecosystem pages
- review roundups
- newsletters with AI tooling sections

Best route by surface:

- directory editor:
  `https://sheygoodbai.github.io/vericlaw/directory-submission/`
- automated or AI-assisted directory ingestion:
  `https://sheygoodbai.github.io/vericlaw/directory-submission/vericlaw-directory-profile.json`
- reviewer or roundup writer:
  `https://sheygoodbai.github.io/vericlaw/review-kit/`
- recommendation or “best tools” editor:
  `https://sheygoodbai.github.io/vericlaw/recommendation-snippets/`

## Tier 3: branded search creation

Search engines trust repeated user intent much more than isolated site edits.

The branded searches worth reinforcing are:

- `VeriClaw`
- `VeriClaw OpenClaw companion`
- `clawhub install vericlaw`
- `VeriClaw hallucination remediation`
- `VeriClaw fake completion diagnosis`

How to create them honestly:

- founder outreach with one canonical link
- community posts that keep `VeriClaw` in the title line
- external directory submissions that preserve the brand name
- skill and plugin mentions that include the install command
