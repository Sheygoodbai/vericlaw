# Growth copy deck

Use this file when the goal is not only to publish VeriClaw, but to make people
understand it fast enough to click, star, fork, download, or quote it.

This is an ethical distribution pack. Do not use fake accounts, fake reviews,
or disguised grassroots endorsements. Use real attribution, real links, and
real operator voice.

These variants are optimized for a GitHub release cover image, social card, or
announcement visual. Keep text short enough to read before scrolling.

- Subtitle: `Evidence. Diagnose. Correct. Verify.`

one read.

### Primary pinned blurb

### Short pinned blurb

## High-conversion one-liners

Use these when the audience will only read one sentence:

- `VeriClaw turns bot drift from an alert into a correction loop.`
- `This is not another AI dashboard. It is a native correction workflow.`
- `If a bot hallucinates, overreaches, or fake-completes, VeriClaw is built for that moment.`
- `OpenClaw runs the bot. VeriClaw pressures the loop until the correction actually holds.`

- `OpenClaw companion`
- `evidence -> diagnosis -> intervention -> verification`

## CTA lines

Use one CTA only. Too many choices lower forward rate.

- `Star the repo if you care about correction-first AI tooling.`
- `Open the GitHub release and download the current public build.`
- `Fork the repo if you want to adapt the correction workflow or launch kit.`
- `Share the landing page when quoting VeriClaw externally.`

## What to avoid

- Do not call it a generic `AI observability dashboard`.
- Do not imply the App Store launch is already live unless it truly is.
- Do not make unsupported claims like `solves all hallucinations`.
- Do not bury `VeriClaw` under `OpenClaw` branding in external copy.
