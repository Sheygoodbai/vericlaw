# External targets

Snapshot date:
`2026-04-06`

This file lists live external submission surfaces that look active enough to
test for `VeriClaw 爪印` right now.

This is not a guarantee of quality or traffic.
It is a practical shortlist of current targets that:

- expose a visible submission route
- appear publicly reachable today
- are broad enough to accept AI tooling, agent tooling, or companion software

## Best first-wave targets

### 1. AIToolsIndex

- Submission URL: `https://aitoolsindex.org/submit`
- Why it is worth trying:
  - visible public submission form
  - simple fields for tool name, URL, description, and email
  - low-friction first-wave test for getting a clean external citation
- Best asset to send:
  - `https://sheygoodbai.github.io/vericlaw/directory-submission/`

### 2. SubmitAITools.org

- Submission URL: `https://submitaitools.org/`
- Why it is worth trying:
  - public AI tools directory with visible free-submission language
  - explicitly references backlinks, search visibility, and AI-agents-style discovery intent
  - broad category coverage that can plausibly fit correction and supervision tooling
- Best asset to send:
  - `https://sheygoodbai.github.io/vericlaw/directory-submission/vericlaw-directory-profile.json`
- Best category guess:
  - `Code & IT` or `Other`

### 3. ToolActive

- Submission URL: `https://toolactive.com/submit-tool`
- Why it is worth trying:
  - public submission instructions are live
  - software-development and AI-tool adjacency is strong
  - manual email submission can sometimes preserve wording better than forms
- Best asset to send:
  - `https://sheygoodbai.github.io/vericlaw/review-kit/`
- Submit path:
  - email `contact@toolactive.com`

### 4. AI Directory

- Submission URL: `https://www.ai-directory.io/`
- Why it is worth trying:
  - active public directory with visible “Submit a tool” route
  - category-oriented tool discovery surface
  - likely useful for brand-name and backlink creation even if traffic quality varies
- Best asset to send:
  - `https://sheygoodbai.github.io/vericlaw/directory-submission/`

### 5. Future AI Guide

- Submission URL: `https://tools.futureaiguide.com/tools/create/`
- Why it is worth trying:
  - public form says submissions are currently free
  - explicit manual review wording
  - the review-heavy posture may help preserve more accurate categorization
- Best asset to send:
  - `https://sheygoodbai.github.io/vericlaw/recommendation-snippets/`

## Routing guidance by target type

### If the target is a general AI tools directory

Lead with:

- directory submission kit
- one-line description:
  `Apple-native correction companion for OpenClaw.`

### If the target is more editorial or review-like

Lead with:

- review kit
- recommendation snippets

### If the target is more OpenClaw- or ecosystem-oriented

Lead with:

- OpenClaw companion page
- ClawHub skill page

## Category guidance

Best categories to try first:

- AI agent correction
- Software development
- Developer tools
- Productivity
- Other

Avoid if possible:

- generic chatbot
- writing assistant
- MCP server
- prompt library

## Success criteria

Treat a target as successful only if at least one of these happens:

- listing goes live with a backlink
- listing preserves `VeriClaw` and the correct one-line description
- listing drives referral clicks
- listing is crawled and becomes searchable later
