# External anchor playbook

Snapshot date:
`2026-04-06`

This file answers one practical question:

When another site links to `VeriClaw 爪印`, what exact anchor text and landing
page should they use so exposure grows without scattering the signal?

## Core rule

Do not spread backlinks evenly across every VeriClaw page.

Use a small routing set:

- brand and install intent -> `https://sheygoodbai.github.io/vericlaw/`
- ClawHub or skill intent -> `https://clawhub.ai/sheygoodbai/vericlaw`
- app intent -> `https://sheygoodbai.github.io/vericlaw/download/`
- Chinese correction intent -> `https://sheygoodbai.github.io/vericlaw/ai-correction-cn/`
- supervision intent -> `https://sheygoodbai.github.io/vericlaw/agent-supervision/`

## Canonical pages by intent

### Brand intent

Use for:

- `VeriClaw`
- `Official VeriClaw`
- `VeriClaw 爪印`
- `VeriClaw plugin`
- `VeriClaw app`

Best URL:

- `https://sheygoodbai.github.io/vericlaw/`

Recommended anchors:

- `VeriClaw`
- `Official VeriClaw`
- `VeriClaw 爪印`
- `VeriClaw official site`

- sending brand anchors to satellite category pages
- using mismatched anchors like `agent evaluation` on the homepage

## Distribution rules

- Use the homepage for brand trust.
- Use the ClawHub skill page for install intent.
- Use one exact-match landing page per weak keyword cluster.
- Do not give every external site a different deep link just because it is available.
- For Chinese long-tail pages, prefer `AI纠偏` first, then `幻觉纠偏`, then `假完成诊断`.
- For English supervision, prefer `Agent Supervision` before broader pages.
