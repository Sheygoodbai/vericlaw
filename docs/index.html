Built for the moment a bot drifts.

VeriClaw is not another monitoring-only surface. It is a correction workspace for teams that need to decide what happened, what should happen next, and whether the fix actually held.

Evidence before guesswork

Capture what the assistant actually did before context disappears into another incident thread or dashboard screenshot.

Role-drift diagnosis

Verification and casebook learning

Close the loop only after the prescribed correction is tested and the reusable lesson is written back into the casebook.

This repository is the public launch pack for the VeriClaw story: screenshots, positioning, release copy, license and attribution materials, and the GitHub-first launch surface.

  2. Diagnose hallucination, overreach, fake completion, or role drift.
  3. Choose an intervention the operator can review and apply.
  4. Verify the correction before calling the loop closed.
  5. Leave behind casebook learning for the next incident.
